# 「最初の1件だけ音声が流れ、2件目以降が流れない」現象の調査

**状況**: report.csv では 99.3% 以上が YES（DB に音声あり）なのに、アプリでは最初に試した音声だけ再生され、手動でパッチしたものは一つも流れない。

---

## 想定される原因（2つ）

### 1. Web のオートプレイ制限（最有力）

**何が起きているか**

- ブラウザでは「ユーザーがタップ/クリックした直後の再生」だけ許可されることがある。
- 「会話全体を再生」で 1 本目は **再生ボタンタップ直後** なので再生される。
- 2 本目は **1 本目の再生が終わったあとのコールバック** から `play()` を呼んでおり、**ユーザー操作の連鎖とみなされない**ため、ブラウザが再生をブロックする。
- そのとき `play()` が `not allowed` で失敗し、`TtsPlaybackBlockedException` が投げられる。
- 現在のコードではこの例外で **ループを break** しているため、**2 本目で止まり「最初の1件だけ」になる**。

**該当コード**

- `lib/services/openai_tts_playback_web.dart`: `play()` 失敗時に `TtsPlaybackBlockedException('not_allowed')` を throw。
- `lib/screens/conversation_study_screen.dart`: `_playAllConversation` 内で `on TtsPlaybackBlockedException { break; }` → ここでループ終了。

**対応**

- Web でブロックされたときは「続きを再生」用の UI を出し、**ユーザーがもう一度タップしたタイミング**で 2 本目以降を再生する（同じループを `startIndex` で再開）。
- これにより「最初の1件だけ」で止まらず、続きを聴けるようにする。

---

### 2. 再生速度（speed）の不一致

**何が起きているか**

- 照合スクリプト `check_tts_prefill_status.dart` は **speed = 1.0** だけで DB を照合している。
- アプリの再生は **設定画面で選んだ再生速度**（`PlaybackSpeedService.getSpeed()`）を `TtsService._defaultSpeechRate` に反映し、**プリフェッチ・再生の両方**でその speed を使う。
- ユーザーが「ゆっくり（0.6）」などにしていると、アプリは **0.6 の cache_key** で検索するが、prefill が **1.0 だけ**なら DB には 0.6 のキーがなく、**2 本目以降はプリフェッチが null → 再生時に DB ミス** になりうる。
- 1 本目だけ流れるように見える場合は、**1 本目だけ先読みで 1.0 がヒットしている**など、たまたま 1 本目だけ条件が合っている可能性がある（環境依存）。

**確認方法**

- 設定で **再生速度を 1.0（標準）** にし、「会話全体を再生」を試す。
- 1.0 で 2 本目以降も流れれば、speed 不一致が原因の可能性が高い。

**恒久対応**

- prefill で **0.6 も投入**する: `dart run scripts/prefill_tts_assets.dart --include-slow`
- これで DB に 1.0 と 0.6 の両方が入り、速度を変えても再生されやすくなる。

---

## まとめ

| 原因 | 現象 | 対処 |
|------|------|------|
| **Web オートプレイ** | 1 本目だけ再生され、2 本目で止まる | ブロック時に「続きを再生」UI を出し、ユーザータップで再開するよう修正する。 |
| **再生速度の不一致** | 速度 0.6 などで 2 本目以降が DB ミス | 設定を 1.0 で試す。恒久は prefill に `--include-slow` で 0.6 も投入。 |

まずは **Web で「続きを再生」で再開できるようにする修正** を行い、それでも 2 本目以降が鳴らない場合は **再生速度 1.0 で試す / prefill --include-slow** を確認する。
