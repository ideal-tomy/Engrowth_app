# 学習者体験の機能改善計画

## Goal

- 初回3分で「音で覚える」体験価値を理解させる。
- 迷ったときは1タップで正しい次アクションに戻れる。
- 毎日の小さな達成が継続と提出につながる。

## Non-Goal

- 学習コンテンツの新規作成（シナリオ・ストーリー）。既存資産の導線・体験改善に注力。

## Exit Criteria

- 60〜90秒で「聞く→話す→返答」を完了可能（Phase 2）。
- 初回導線から迷子にならずホームに戻れる。
- チュートリアル離脱率が改善され、KPIで検証可能。

---

## 1. 初回体験（オンボーディング・チュートリアル）

### 1.1 現状

- OnboardingFlowScreen: 7ステップ
- TutorialConversationScreen: 挨拶体験（事前生成音声）
- 使い方/コンセプトページあり

### 1.2 改善方針

- 事前生成音声による低遅延チュートリアルを維持・強化。
- 離脱ポイントの可視化と導線文言・順序の反復改善。
- 初回導線完了後、ホームに自然に誘導する。

### 1.3 参照

- TUTORIAL_SCHEMA_DESIGN.md
- TUTORIAL_FUNNEL_QA.md
- DAILY_VOICE_ONBOARDING_QA.md

## 2. 学習導線

### 2.1 会話トレーニング

- 30秒/3分選択 → シナリオ学習/ストーリー学習
- 録音→練習保存→提出の導線を明確化。
- ShareBoundaryNotice で「提出済みのみコンサルに共有」を常時説明。

### 2.2 再開・継続

- ResumeLearningCard: 1タップで続きから起動可能に（Phase 3）。
- RecommendedCard: 次アクションの明確化。

### 2.3 振り返り

- 録音履歴: 練習/提出済みタブで区別。
- 提出前確認モーダル: 共有相手・取り消し不可を明示。

---

## 3. 設定・アカウント

- ヒント設定（2秒）、再生速度、通知一覧の整理。
- 匿名→アカウント作成の導線をスムーズに。
