# 全画面デザインアップグレード方針

## Goal

- 全画面で一貫したデザイントークン・演出・コンポーネントを適用する。
- MASTER_PLAN の Sound-First、Zero-Latency、Consultant Bridge に沿った視覚的体験を構築する。
- ライト/ダーク双方で可読性とUI統一を維持する。

## Non-Goal

- デザインシステムの大規模再設計。既存 `ENGROWTH_DESIGN_SYSTEM.md` をベースに拡張・統一する。
- 個別画面の詳細レイアウト設計（それは各機能計画で扱う）。

## Exit Criteria

- トークン・演出・一貫性の方針が文書化され、新規画面作成時の参照として使える。
- engrowth_theme.dart 準拠が明文化され、主要画面のカラー・余白・タイポがテーマ準拠となる。

---

## 1. デザイントークン

### 1.1 既存ベース

`docs/ENGROWTH_DESIGN_SYSTEM.md` および `lib/theme/engrowth_theme.dart` を唯一の基準とする。

| 用途 | 値 | 備考 |
|------|-----|------|
| プライマリ | #d30306 | CTA、プログレス、ハイライト |
| 背景 | #f5f5f5 | オフホワイト |
| サーフェス | #ffffff | カード・ダイアログ |
| スペーシング | 8px ベース | xs=4, sm=8, md=16, lg=24, xl=32, xxl=48 |

### 1.2 統一ルール

- 新規ウィジェットは `Theme.of(context).colorScheme` を参照する。ハードコード色は禁止。
- 角丸・余白は設計システムのトークンを使用する。
- ライト/ダーク切替時も colorScheme に委ね、個別フォールバックを設けない。

---

## 2. 演出基準（Dopamine Loop）

MASTER_PLAN Phase 3 の「達成感の爆上げ」に沿う。

### 2.1 学習完了時

- リザルト演出: カウントアップ、成功音、ハプティクスを統一。
- 触感: HapticFeedback.mediumImpact / heavyImpact を重要アクションに適用。

### 2.2 ボタン・カード

- タップ時: スケール 0.95 → 1.0（200ms）
- カード: エレベーション（影）変化で押下感を表現。

### 2.3 待ち・思考中

- オートアドバンスでタップ回数を最小化。
- 思考中アニメーションで自然な「溜め」を設計（Zero-Latency 原則維持）。

---

## 3. コンポーネント一貫性

### 3.1 共通化対象

- カード系: 背景・角丸・パディングの統一
- CTA ボタン: プライマリカラー、サイズ、タップフィードバック
- プログレスバー: 再生中フレーズ単位のアニメーション
- リスト行: タップ領域、区切り線、アイコンサイズ

### 3.2 画面別ガイドライン（継承）

- 会話学習: 中央の背景イラストを隠さない。CTA は下部・プライマリ統一。
- 一覧系: カードレイアウト、サムネイル優先、テーマタグはプライマリの薄いバリアント。

---

## 4. ホーム導線の視覚優先度

- 主要導線は初見3秒で認識可能とする（Phase 1 Exit Criteria）。
- 1画面1目的。説明過多を避け、親指ゾーンに主要操作を配置。
