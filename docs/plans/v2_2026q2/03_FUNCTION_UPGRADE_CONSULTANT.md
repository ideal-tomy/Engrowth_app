# コンサル導線・評価体験の機能改善計画

## Goal

- 学習データを伴走品質に変換する運用基盤を完成させる。
- 提出からフィードバックまでの運用導線を確立する。

## Non-Goal

- 新規ビジネスロジックの大規模追加。既存 UI・サービス層の完成度向上に注力。

## Exit Criteria

- 提出からフィードバックまでの運用導線が確立する。
- KPI（未対応・本日対応・担当数）が実運用で参照可能。
- 課題発行タブがプレースホルダーから実機能へ進化する（範囲内で）。

---

## 1. 現状

### 1.1 判定

- consultant_assignments に consultant_id で登録、または devViewAsConsultantProvider。

- KPI カード（未対応・本日対応・担当数）
- 詳細ボタン（ドロワー、現状プレースホルダー or 実データ）

### 1.3 TODO

## 2. 改善方針

### 2.1 提出レビュー高速化

- 再生・テンプレ・コメント・ステータス更新の一貫した UX。
- キーボードショートカットやジェスチャーによる効率化（将来検討）。

### 2.2 詳細ログ

- user_sessions 導入後に実データ表示を本格化。
- 現状はプレースホルダーで構造を維持。

### 2.3 課題発行

- 課題発行タブをプレースホルダーから実機能へ段階的に進化。
- プリセット選択・カスタム文言の両対応を検討。
