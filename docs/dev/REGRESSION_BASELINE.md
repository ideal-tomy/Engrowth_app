# 回帰確認基準（Phase 0）

リファクタリング前後の挙動差を比較するための基準を記録します。各PRのマージ前に、この手順で確認を行ってください。

## 1. 分析・ビルド確認

### flutter analyze
```bash
flutter pub get
flutter analyze
```
- **合格条件**: エラー 0、警告は既存のもののみ（新規警告を増やさない）

### Webビルド
**Windows (PowerShell):**
```powershell
.\scripts\build_for_deploy.ps1
```

**macOS/Linux:**
```bash
./scripts/build_for_deploy.sh
```

- **前提**: `.env` に `SUPABASE_URL` と `SUPABASE_ANON_KEY` が設定済み
- **合格条件**: `flutter build web --release` が正常終了し、`build/web` が生成される

## 2. ローカル起動確認

```bash
flutter run -d chrome
```

- **合格条件**: アプリが起動し、初期画面（`/home`）が表示されること

## 3. 主要導線のスモークテスト

以下の項目をリファクタリング前後で同等に動作することを確認します。

### 3.1 認証

| 項目 | 確認方法 | 合格条件 |
|------|----------|----------|
| 匿名ログイン | 起動後、ダッシュボードを表示 | 自動で匿名ログインされ、エラーなく表示される |
| ログイン済み画面（開発用） | 設定メニューで「開発: ログイン済み画面」をオン | バナー非表示・統計等が表示される（kDebugMode時のみ） |
| ログアウト後 | ログアウト実行 | 自動で再匿名サインインし、問題なく利用できる |

参照: [docs/ANONYMOUS_AUTH_SETUP.md](../ANONYMOUS_AUTH_SETUP.md)

### 3.2 画面遷移

| ルート | 遷移元 | 合格条件 |
|-------|--------|----------|
| /home | 初期・タブ | ダッシュボードが表示される |
| /library | タブ | Library Hub（シナリオ・会話・3分ストーリー等）が表示される |
| /progress | タブ | 進捗画面が表示される |
| /words | タブ | 単語一覧・検索が表示される |
| /study | ダッシュボード等 | 瞬間英作文学習が開始できる |
| /scenarios | Library | シナリオ一覧が表示される |
| /conversations | Library | 会話一覧が表示される |
| /story/:id | Library | 3分ストーリー学習が表示される |
| /account | 設定等 | アカウント画面が表示される |

### 3.3 学習進捗

| 項目 | 確認方法 | 合格条件 |
|------|----------|----------|
| 瞬間英作文 | Quick30/Focus3 で進捗 | 進捗が保存され、次回起動時に反映される |
| シナリオ学習 | シナリオで会話を再生・進捗 | 進捗が保存され、進捗画面に反映される |
| 会話学習 | 音声再生・録音・役練習 | 問題なく再生・録音・比較ができる |
| 3分ストーリー | ストーリー表示・続きから再開 | 進捗保存と続きから再開ができる |

### 3.4 音声

| 項目 | 確認方法 | 合格条件 |
|------|----------|----------|
| TTS再生 | 学習画面で再生ボタン | 音声が再生される |
| 再生速度 | 設定で速度変更 | 変更が反映される |
| 録音・比較 | 会話学習で録音 | 録音・再生・比較ができる |

## 4. デプロイ確認

本番相当のビルドが正常に完了し、デプロイ可能であること。

```powershell
# Windows
.\scripts\deploy_firebase.ps1
```

```bash
# macOS/Linux
./scripts/deploy_firebase.sh
```

- **合格条件**: `firebase deploy --only hosting` が正常終了し、デプロイ後のURLでアプリが動作する
