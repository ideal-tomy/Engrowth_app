# 英会話品質ルーブリック（Conversation Quality Rubric）

第二言語習得（SLA）の科学的見地に基づき、Engrowthアプリの英会話コンテンツを評価・分類するための基準を定義します。

---

## 1. 評価軸とスコアリング

| 軸 | 説明 | 閾値（OK判定） |
|----|------|----------------|
| **Naturalness** | 米国日常・職場で自然に使われる表現か | ネイティブが違和感なく言い得る |
| **Pragmatics** | 丁寧さ、距離感、場面適合が適切か | 文脈に合った敬度・トーン |
| **Learnability** | 初級1000語/中級3000語で理解可能か | 指定語彙内で成立 |
| **Speakability** | 音読しやすさ、息継ぎ、リズムが自然か | 15語以内のチャンクで区切れる |
| **Transferability** | 別シーンへ転用可能な頻出表現か | 汎用フレーズ・チャンクである |

各発話を 1〜5 点で採点し、全軸 4 以上を **OK**、いずれか 3 以下を **要修正**、意図不明・文法崩壊を **全面改稿** とする。

---

## 2. 分類定義

### OK（そのまま利用可）
- 全軸 4 以上
- 米国日常会話で実際に使われる表現
- 例: "Hello. I would like to check in, please." / "These pants are a bit long. Do you offer hemming services here?"

### 要修正（意図を保持して自然な英語へ書換）
- 意味は通じるが不自然な直訳
- 語彙の誤用（establish→set up, direction→instructions, reveal→show）
- 文法的には正しいが、口語として不自然
- 例: "We establish your user name" → "We'll set up your username from the form"

### 全面改稿（意図を再確認の上で新規執筆）
- 単語の羅列のみ（文として成立していない）
- 日本語の逐語訳で英文として崩壊
- 例: "Production. Weight of symptoms. I perform rest." → 医師の指示として自然な一文に書き直す

---

## 3. 典型的な問題パターン

| パターン | 例 | 修正方針 |
|----------|-----|----------|
| 語彙の誤用 | establish（設立）を「設定」の意で使用 | set up, create, enter |
| 語彙の誤用 | direction（方向）を「手順」の意で使用 | instructions, steps, next step |
| 語彙の誤用 | reveal（暴露）を「表示」の意で使用 | show, display |
| 語彙の誤用 | production（生産）を文脈外で使用 | 削除または適切な語に置換 |
| 語彙の誤用 | perform rest（休息を遂行） | get some rest, rest up |
| 直訳崩壊 | weight of symptoms（症状の重さ） | severity of symptoms |
| 単語羅列 | "Email. Professional. Legal. Form. Done." | 完整な文に再構成 |
| 過剰な目標語彙詰め込み | 1000語リストの語を無理に複数挿入 | 自然な会話を優先し、語彙は結果的に抽出 |

---

## 4. SLA設計原則（品質判断の背景）

- **Comprehensible Input + i+1**: 既習語彙の上に少量の新規語彙を追加。理解可能な入力を維持。
- **Formulaic Sequences**: 頻出チャンク（"I'd like to ~", "Could you tell me ~"）を優先。
- **Pushed Output**: 学習者が発話しやすい長さ・難度。
- **Negative Evidence の回避**: 不自然な例を覚えさせない（誤った入力は習得を阻害）。

---

## 5. シーン別の期待トーン

| シーン | 期待トーン | 備考 |
|--------|-----------|------|
| 挨拶・自己紹介 | 親しみやすく丁寧 | ビジネス/学生で敬度を調整 |
| 道案内 | 簡潔で親切 | 短い文で区切る |
| 空港・ホテル | 丁寧で効率的 | 定型表現を多用 |
| カフェ・レストラン | カジュアル〜丁寧 | 店員と客の役割明確 |
| 病院 | 冷静で具体的 | 症状説明は簡潔に |
| 銀行・郵便局 | 丁寧で正確 | 専門用語は最小限 |
| ビジネスメール・プレゼン | フォーマルで明確 | 文語寄りも許容 |

---

## 6. 採点チェックリスト（1発話あたり）

- [ ] 英文として文法的に成立しているか
- [ ] ネイティブが日常で使い得る表現か
- [ ] 日本語訳と意味が対応しているか
- [ ] 目標語彙レベル（1000/3000語）に収まっているか（結果的に）
- [ ] 15語以上の長文は適切に区切れるか
- [ ] 語彙の誤用（establish/direction/reveal/production 等）がないか
- [ ] 単語の羅列になっていないか
