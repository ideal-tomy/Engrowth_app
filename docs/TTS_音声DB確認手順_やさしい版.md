# 「いまDBに入ってる音声、本当に再生できる？」を確認する手順（やさしい版）

**だれ向け**: キャッシュキーとかよくわからない人。  
**やること**: いまDBに保存されている音声が、アプリで再生できる状態かどうか確認する。

---

## おおまかな流れ（3ステップ）

1. **一覧を出す** … どの発話が「DBにある」か一覧にする
2. **一覧を見る** … YES＝再生できる想定、NO＝まだない
3. **1つだけ詳しく見たいとき** … その発話のキーがDBと合ってるか確認する

---

## ステップ1: 一覧を出す

**やること**: 全部の発話について「DBに音声があるか」を調べて、CSVファイルに書き出す。

1. パソコンで **PowerShell** または **ターミナル** を開く
2. このアプリのフォルダ（`engrowth_app`）に移動する  
   - 例: `cd C:\Users\ryoji\demo\engrowth_app`
3. 次のコマンドを **そのまま** 打って Enter:
   ```
   dart run scripts/check_tts_prefill_status.dart --output=report.csv
   ```
4. 終わるまで待つ（発話数が多いと少し時間がかかる）
5. 同じフォルダに **`report.csv`** というファイルができる

**ここでやってること**:  
アプリが使うのと **同じルール（キャッシュキー）** で、ひとつひとつ「DBにそのキーがあるか」を照合している。だから「一覧の結果＝アプリで再生できるかどうか」と対応している。

---

## ステップ2: 一覧を見る

**やること**: `report.csv` を開いて、どの発話が「再生できる」かを見る。

1. **report.csv** を Excel やメモ帳などで開く
2. いちばん右の列 **`in_tts_assets`** を見る:
   - **YES** … その発話の音声はDBに入っている → **アプリで再生できる想定**
   - **NO** … その発話の音声はDBにない → **アプリでは鳴らない想定**

**まとめ**:
- **YES の数** = 「いまDBにあって、再生できる想定の数」
- **NO の数** = 「DBにない（またはキーが違う）ので、鳴らない想定の数」

「キャッシュキーを合わせて」の意味は、このスクリプトが **アプリと同じキーで照合している** ので、この一覧の YES/NO がそのまま「現時点でDBに保存されている音声が再生できるか」の答え、ということ。

---

## ステップ3: 1つだけ「この発話、本当にDBにある？」と確認したいとき

**やること**: 気になる英語のセリフを1つ指定して、そのキーがDBにあるか確認する。

1. PowerShell / ターミナルで、アプリのフォルダにいる状態にする
2. 次のコマンドの **「ここに英語を入れる」** のところを、調べたい発話の英語に変えて実行:
   ```
   dart run scripts/verify_tts_cache_hash.dart --text "ここに英語を入れる"
   ```
   例:
   ```
   dart run scripts/verify_tts_cache_hash.dart --text "Hello. I want to cancel my card immediately."
   ```
3. 出てきたメッセージを見る:
   - **「✅ DB にこの cache_key が存在します」** … その発話（話者 nova として）はDBに入っている → 再生できる想定
   - **「❌ DB にこの cache_key がありません」** … そのキーではDBにない。  
     - 会話で **話者A** のときは voice が **alloy** になるので、このコマンド（nova 固定）では「ない」と出ることがある。  
     - 一覧（report.csv）では話者ごとにキーを合わせているので、一覧の YES/NO のほうが本命。

**注意**: このコマンドは「新規に音声を作る」ために OpenAI を呼ぶ場合があり、そのとき API の残高（429）でエラーになることがある。**「DBに存在するか」の判定だけなら、上の「✅ DB にこの cache_key が存在します」が出れば十分**で、その下のエラーは無視してよい。

---

## 用語の意味（こども向け）

| ことば | 意味（やさしく） |
|--------|------------------|
| **DB** | 音声の「名前」と「どこにファイルがあるか」を覚えておく箱。アプリはここを見て音声を探す。 |
| **キャッシュキー** | その音声を一意に指す「名前」のルール。同じテキスト・話者・速度・モデルなら同じキーになる。 |
| **キーを合わせる** | アプリが「このキーで探す」と言っているのと、DBに保存されているときの「名前」を同じにすること。合わせないと「あるのに見つからない」になる。 |
| **in_tts_assets = YES** | 「そのキーで探したら、DBにあった」＝いまの設定なら再生できる想定。 |
| **in_tts_assets = NO** | 「そのキーで探したら、DBになかった」＝いまの設定では鳴らない想定。 |

---

## まとめ（一言で）

- **一覧を出して** `report.csv` の **in_tts_assets** を見れば、**キャッシュキーを合わせたうえで「現時点でDBに保存されている音声が再生できるか」** が分かる。
- **YES** = 再生できる想定、**NO** = 鳴らない想定。
- 1発話だけ詳しく見たければ、`dart run scripts/verify_tts_cache_hash.dart --text "その英語"` で「DB にこの cache_key が存在します」かどうかを確認できる。
