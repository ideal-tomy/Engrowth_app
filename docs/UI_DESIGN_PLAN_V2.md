# Engrowth アプリ UI設計計画書 v2

## 概要

Engrowth が提供する英会話学習アプリの UI/UX を、優れた事例を参考に再設計するための包括的な計画書です。

## 1. 参考とする優れた UI 事例

### 1.1 カラオケまねきねこ

**参考ポイント**
- **機能別タブ構成**: 検索・ランキング・マイページなど、機能が明確に分かれたタブ UI
- **直感的なアイコン**: 絵文字・シンプルアイコンで迷わず操作できる
- **親しみやすいキャラクター**: まねきねこによる雰囲気づくり
- **コンパクトな情報配置**: 1画面で完結し、迷いのないフロー

**Engrowth への適用**
- 会話学習・単語学習・設定などの主要機能をタブまたはドロワーで整理
- 聴く / A役 / B役 など、役割をアイコンで直感的に表現（既存の実装を強化）
- シチュエーションイラストや親しみやすいビジュアルで学習意欲を高める

### 1.2 Duolingo

**参考ポイント**
- **ゲーミフィケーション**:  streak（連続日数）、XP、リーダーボード
- **進捗の見える化**: 円形プログレス、レッスン完了のチェック
- **キャラクター**: ふくろうなど親しみやすいマスコット
- **継続しやすい UX**: 1レッスン短時間、達成感を与える演出

**Engrowth への適用**
- 会話学習の進行状況をプログレスバーで明確に表示（実装済み）
- 週間学習回数・連続学習日数の表示を検討
- 達成時の軽いアニメーションやフィードバック

### 1.3 その他英語アプリ（シャドテン / スピークバディ 等）

**参考ポイント**
- **習慣化のサポート**: 目標達成の可視化、リマインダー
- **音声中心 UI**: 再生ボタンが目立ち、テキストは補助的
- **段階的ヒント**: 思い出せないときの足場かけ（LEARNING_MODE_DESIGN.md と連動）

## 2. Engrowth ブランドカラー

| 用途         | 色コード   | 説明           |
|--------------|------------|----------------|
| プライマリ   | `#d30306`  | ブランドレッド |
| 背景         | `#f5f5f5`  | オフホワイト   |
| サーフェス   | `#ffffff`  | 白（カード等） |
| A役（お客様）| `#1565C0`  | 青系           |
| B役（店員）  | `#2E7D32`  | 緑系           |

プライマリは CTA ボタン・プログレスバー・ハイライトに使用し、一貫したブランド感を維持します。

## 3. 現在のアプリ機能と UI 方針

### 3.1 会話学習

| 機能             | 方針                                                 |
|------------------|------------------------------------------------------|
| シチュエーション | 背景画像を中心に配置し、視認性を最優先               |
| 音声ファースト   | 初期はテキスト非表示、再生後に表示（VOICE_FIRST 準拠）|
| プレースホルダー | 「会話全体を聞く」の直上に配置し、中央画像を邪魔しない |
| モード選択       | 聴く / A役 / B役 をアイコン＋ラベルで明示            |
| プログレス       | 再生中はゆっくり進行するアニメーション付きバー       |

### 3.2 シチュエーション学習（例文・単語）

- シチュエーション背景＋ヒントフェード（LEARNING_MODE_DESIGN.md 準拠）
- テーマカラーを AppBar・ボタンに適用

### 3.3 ナビゲーション

- スクロール不要の 1 画面レイアウトを基本
- 下部タブ or ドロワーで主要機能へ遷移

## 4. 画面別デザインガイドライン

### 4.1 会話学習画面

```
┌─────────────────────────────────┐
│ [×]           会話の進行  [👁]   │
│ ▓▓▓▓▓▓░░░░ 1/10                 │  ← Engrowth赤でプログレス
├─────────────────────────────────┤
│                                 │
│    [背景イラスト：全面表示]      │
│                                 │
│         [A（お客様）] のみ       │  ← 音声未再生時は最小限
│         （再生中: 再生中バッジ）  │
│                                 │
├─────────────────────────────────┤
│ 音声を再生するとテキストが…     │  ← 「会話全体を聞く」の直上
│ ┌─────────────────────────────┐ │
│ │  会話全体を聞く              │ │  ← プライマリ色の CTA
│ └─────────────────────────────┘ │
│ [A役として再度挑戦][B役として…] │
│ [この発話を再生] [<] [>]        │
└─────────────────────────────────┘
```

- **中央**: 背景イラストを隠さない（プレースホルダーは下部へ移動済み）
- **CTA**: プライマリ（#d30306）で統一
- **役ボタン**: A役=青系、B役=緑系で区別

### 4.2 会話一覧（シチュエーション選択）

- カードレイアウト、サムネイルを大きく
- テーマタグはプライマリの薄いバリアント
- 聴く / A役 / B役 ボタンは Engrowth テーマで統一

### 4.3 学習モード画面

- AppBar: プライマリ背景
- ヒントフェードはプライマリの薄いトーンで光る演出
- 「覚えた」ボタンはプライマリ色で目立たせる

### 4.4 シナリオ学習ページ（新規）

- **レイアウト**: セクション + 横スクロール行（Netflix型）
- **セクション**: カフェ / ホテル / 空港 などシチュエーション別
- **カード**: サムネイルを大きく、タップで会話学習画面へ直接遷移
- 詳細は [SCENARIO_LEARNING_PAGE_IMPLEMENTATION.md](./SCENARIO_LEARNING_PAGE_IMPLEMENTATION.md) を参照

## 5. タイポグラフィ・スペーシング

UI_UX_DESIGN.md の内容を継承しつつ、以下を追加：

- **見出し**: 太字、適切な余白
- **本文**: 読みやすい行間（1.4〜1.5）
- **8px ベース**: xs=4, sm=8, md=16, lg=24, xl=32

---

## 6. アニメーション・フィードバック

- **プログレスバー**: 再生中はフレーズ単位でスムーズに進行（実装済み）
- **ボタンタップ**: 軽いスケール＋触覚フィードバック
- **ヒント表示**: 300ms フェードイン（LEARNING_MODE_DESIGN 準拠）
