# Speak風UX原則 — アプリ全体の指針

**Cursor / AI 向け**: 本ドキュメントはユーザー向け画面の UI/UX 設計の**一次資料**。実装・判断の際は必ず参照。`.cursor/rules/speak-style-ux-app-wide.mdc` と整合。

ユーザーが操作する画面（コンサルタント・管理者ダッシュボードを除く）は、本ドキュメントの原則に従い、「静寂と動きの同期」「視覚説明への依存最小化」「ゆっくりした自然な流れ」を実現する。

**参照:** `MASTER_PLAN.md` の哲学と Phase と整合。本ドキュメントは UI/UX の「どう見せる・どう動かすか」の具体指針。

## 2. 目指す体験（完成形）

- **映画のカット割りのように**「静寂（余白）」と「動き（アニメーション）」がはっきり分かれ、一つの流れとして同期している。
- **ゆっくり**は「遅い」ではなく、意図した**余韻・余白**。あえて時間をかけて、達成感や理解の定着を促す。
- **説明は最小限**。テキストや「ボタンを押して説明を見る」に頼らず、**自動で表示・切り替え**される流れで理解できるようにする。
- **ボタン操作やページ移動**は、ユーザーに「押して」「次へ」と促すのではなく、**ゆっくりした流れの中で自然に次のアクションに誘導**する（Speak のような導線）。

---

## 3. 設計原則

### 3.1 静寂と動きの同期（Sequence）

- **シークエンス**：「Aが終わったらB、Bが終わったらC」を一つの振付として設計する。単発のアニメーションの羅列にしない。
- **余韻（余白）**：重要なアクションや音声の直後に、即遷移せず **1〜2秒程度の待ち** を入れる。
- **遷移**：標準の素早いスライドではなく、**1秒以上のクロスフェード** など、意図した長さのトランジションを使う。

### 3.2 視覚情報への依存を減らす

- **説明テキスト**：長文の説明は避ける。必要な場合は短く、段階的に自動表示する。
- **ポップアップ・ダイアログ**：「ボタンを押すと説明が出る」より、**流れに合わせて自動で表示・切り替え**する。
- **案内**：どこをタップするかは、**ハイライト・波紋・時間差での表示**など、動きで示す。
- **行動要請の最小化（Speak 参照）**：**ユーザーへの行動要請は、音声再生以外は原則不要**。テキストで「〇〇してください」と促すより、自然の流れで把握しながら自動で学習のための再生まで流れるように設計する。

### 3.3 自動進行と自然な誘導

- **自動で動く体験**：デモや導線では、Timer / AnimationController の完了をトリガーに、**自動で次の演出や遷移**を行う。
- **ボタン・CTA**：ユーザーに「押して」と強く促すより、**流れの中で自然に押したくなるタイミング**（余韻の後、説明の後など）を設計する。
- **ページ移動**：「次へ」を連打させるのではなく、**一連の流れの終わりに自然に次の画面へ**移る。

### 3.4 技術的な実装方針

- **Chaining Animations**：アニメーションを鎖のようにつなぐ。`Future.delayed` と `AnimationController` で「待ち → アニメ → 完了 → 次」を明示する。
- **Manual Controller**：必要に応じて `AnimationController` を自前で定義し、`forward()` で制御する。「いつ何が終わったか」をコード上で明確にする。
- **ポップアップの「ゆっくり」**：背景ぼかし（BackdropFilter）も **TweenAnimationBuilder** などで 0→目標値 にアニメーションし、その後にテキスト窓を時間差で表示する。
- **遷移**：`PageRouteBuilder` や GoRouter の CustomTransitionPage で、**クロスフェードの長さ** を 1秒以上に設定する。

## 4. 振付のテンプレート（1ステップの流れ）

一つの説明やデモの単位は、次のフェーズで設計する。

| フェーズ | 内容 | 目安 |
|----------|------|------|
| **登場** | 背景ぼかし＋コンテンツのふわっと表示 | 約1秒 |
| **滞在** | ユーザーが読む・理解する時間 | 2〜3秒 |
| **演出** | デモ（ボタン光る・擬似タップなど） | 必要に応じて |
| **余白** | 完了後の無音・待機（余韻） | 1〜1.5秒 |
| **退場** | フェードアウト → 次ステップへ | 約1秒 |

全体を **TutorialSequencer のような一連の流れ** として実装し、`Future.delayed` と `AnimationController` で制御する。

## 5. 既存資産との関係

- **Motion Token**：`lib/theme/engrowth_theme.dart` の `EngrowthRouteTokens` / `EngrowthElementTokens` を優先して使う。チュートリアル・ゆっくり遷移用の長めの値（例: `tutorialCrossfadeDuration`）を、ユーザー向け導線で一貫して使う。
- **微調整**：長さ・イージングの変更は `docs/MOTION_TUNING_GUIDE.md` の「このファイルのこの数字」に従う。
- **MASTER_PLAN**：Sound-First・Zero-Latency は維持する。**意図した余韻・余白** は「贅沢な時間」として仕様に含め、無駄な待ちとは区別する。

## 6. 開発時のチェックリスト

- [ ] 説明は自動表示・自動切り替えで足りるか？ ボタン押下前提の説明になっていないか？
- [ ] 「〇〇してください」などの行動要請テキストは不要か？ 音声再生以外のユーザー操作を促していないか？
- [ ] 重要なアクションの直後に余韻（1秒以上）を入れているか？
- [ ] ページ遷移は 1秒以上のクロスフェード（または同等のゆっくりしたトランジション）か？
- [ ] ポップアップは「ぼかしのアニメーション ＋ 時間差でコンテンツ表示」になっているか？
- [ ] 「Aが終わったらB」が、Timer / AnimationController の完了で明示的に制御されているか？
