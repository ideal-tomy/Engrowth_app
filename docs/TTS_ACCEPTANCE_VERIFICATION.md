# TTS 自然会話 検証シナリオ（PLAN 完了定義）

デプロイ後TTS自然会話改善PLAN の合格基準を確認するための検証シナリオ。

## 前提

- `tts_synthesize` Edge Function がデプロイ済み
- prefill を実行済み（キャッシュヒット率向上のため）
- `analytics_events` テーブルで `tts_playback_session`, `tts_request` イベントを確認可能

## ケースA: 全ヒット想定（2回連続再生）

**手順:**
1. 会話学習画面を開く
2. 「会話を聴く」をタップ
3. 会話全体の再生完了を待つ
4. ダイアログで「もう一度全体を聴く」をタップ

**確認:**
- 2回目の `tap_to_first_audio_ms` が 700ms 以下であること
- `utterance_gap_avg_ms` が 350ms 以下であること（計測ログ）
- 無音の長時間停止が発生しないこと

## ケースB: ミス混在（新規発話を含む会話）

**手順:**
1. prefill 未実施の会話、または新規追加した発話を含む会話を開く
2. 「会話を聴く」をタップ

**確認:**
- 最初の1フレーズで遅延があっても「準備中」的な体感（プログレス表示など）があること
- 無音のまま長時間停止してしまう状態が発生しないこと
- 連続再生が途切れず最後まで再生されること

## ケースC: モバイル実機（iOS）

**手順:**
1. iPhone 実機でアプリをビルド・起動
2. 会話学習画面で「会話を聴く」をタップ
3. 連続再生 → 途中で停止 → 再開を実施
4. 役割モード（A役/B役）に切り替えて会話を聴く

**確認:**
- 1フレーズだけ流れて止まったように見える症状が再現しないこと
- 停止・再開で古い再生が混線しないこと

## ケースD: 役割モード（roleA / roleB）

**手順:**
1. 会話を聴く完了後、A役またはB役を開始
2. 相手の番の TTS 再生 → 自分の番の録音待機の流れを確認
3. 役割切替（A→B または別会話）を実施

**確認:**
- 役割切替時に古い TTS 再生が混線しないこと
- 相手の番の TTS が自然な間で再生されること

## 計測の確認

Supabase Dashboard → `analytics_events` で以下を確認:

| イベント | 主なプロパティ |
|----------|----------------|
| `tts_playback_session` | `conversation_id`, `first_utterance_prefetched`, `tap_to_first_audio_ms`, `prefetch_hit_count`, `prefetch_miss_count` |
| `tts_request` | `latency_ms`, `cache_hit`, `tts_session_id` |

Edge Function ログ（`cache_hit`, `cache_key_hash`, `voice`, `speed`, `text_length`）でキャッシュの状況を追跡可能。

## 合格基準（Done）

- [ ] 初回再生の体感遅延が改善し、「押しても無音が続く」報告が解消
- [ ] 1フレーズ停止感の再現率が大きく低下し、連続再生の自然さが確認できる
- [ ] キャッシュ命中率・再生指標がダッシュボード/ログで継続監視できる
