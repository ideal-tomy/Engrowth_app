# ナビゲーション/UX洗練（M3/状態保持）導入手順

## 目的
スマホでの操作性を最大化し、学習導線を最短化する。

## 対象機能
- M3 `NavigationBar`への統一
- タブ切り替え時の状態保持
- 初期画面の学習優先化

## UI/UX要件（スマホ前提）
- メイン4機能は常時表示
- 親指で届く位置に主要CTA
- タブ切替後もスクロール/フィルタ状態を維持

## 技術要件
### 1) go_router 状態保持
`StatefulShellRoute` で各タブ状態を保持する
- タブ遷移で画面がリセットされない
- Deep Link対応

### 2) M3 NavigationBar
- 既存BottomNavを置き換え
- selected/labelのデザインを統一

## 実装手順
1. **ルーティング整理**
   - `StatefulShellRoute`採用
   - 各タブのルートをブランチ化
2. **UI差し替え**
   - `BottomNavigationBar` → `NavigationBar`
3. **初期導線**
   - `/` → `/study`へリダイレクト
4. **状態保持確認**
   - タブ切替時にスクロール位置維持

## 受け入れ条件
- タブ切替で学習状態が保持される
- M3ナビでデザイン統一される
- アプリ起動後すぐ学習開始できる
