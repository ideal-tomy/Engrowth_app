# Marquee導線 比較検証チェックリスト

フラグON/OFFで既存UIとMarquee導線を比較する際の観点です。

## フラグ切替方法

1. アプリ起動後、ホーム画面でハンバーガーメニュー（左上）を開く
2. デバッグモード時のみ「Marquee導線」のスイッチが表示される
3. ON: ヘッダー・フッター上に流れるタブを表示
4. OFF: 既存UI（変更なし）

## 比較観点

### 1. 表示崩れ

- [ ] フラグOFF時、既存レイアウトがmainブランチと同一である
- [ ] フラグON時、ヘッダーMarqueeがヘッダー直下に正しく表示される
- [ ] フラグON時、フッター上レールがナビバーの上に正しく表示される
- [ ] 縦スクロール時、ヘッダーMarqueeが画面上部に固定表示される（スクロールに追従）
- [ ] フッター・レールが画面下部に固定表示される
- [ ] キーボード表示時、レイアウトが崩れない
- [ ] 端末回転時、レイアウトが崩れない

### 2. 操作性

- [ ] ヘッダーMarqueeのタブをタップすると、正しい画面に遷移する
- [ ] フッターレールのタブをタップすると、正しい画面に遷移する
- [ ] ヘッダーMarqueeを横スクロールすると、自動スクロールが一時停止する
- [ ] スクロール終了後、約1.2秒で自動スクロールが再開する
- [ ] タップ後、該当タブが60秒間非表示になる（抑制）
- [ ] ボトムナビのタップ領域が十分である（44dp以上）

### 3. 導線効果

- [ ] 未ログイン時、適切な文言（30秒会話、3分会話、アカウント作成等）が表示される
- [ ] ログイン直後（7日以内）、目標・再開・復習等の文言が表示される
- [ ] 継続ユーザー時、再開・復習・お気に入り等の文言が表示される
- [ ] 動的項目（前回の続き、今日の目標、本日の復習）が状態に応じて表示される

### 4. アクセシビリティ

- [ ] システムの「動きを減らす」設定時、自動スクロールが無効になる
- [ ] タブがスクリーンリーダーで読み上げられる

### 5. 計測

- [ ] タップ時に `marquee_tap` イベントが送信される（analytics_events テーブル）
- [ ] イベントに `source` と `label` が含まれる

## スクリーンショット比較

以下の状態でスクリーンショットを取得し、mainブランチと比較することを推奨します。

1. 初回起動時（未ログイン）
2. スクロール中（ヘッダーMarqueeが表示されている状態）
3. キーボード表示時（検索画面等）
4. フラグOFF時（既存UI）
