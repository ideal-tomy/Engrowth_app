# キャッシュを合わせる方法（やさしい版）

**だれ向け**: キャッシュキーってなに？ どうしたら「合う」の？ という人。  
**やること**: 「音声の名前のルール」を、アプリ・事前準備・サーバーで **ぜんぶ同じ** にすること。

---

## ① キャッシュってなに？（たとえ話）

- **キャッシュ** = いったん作った音声を **しまっておく箱** と、その箱に貼る **名前シール** のこと。
- アプリは「この名前の音声ください」と言って探す。名前が **ぴったり同じ** じゃないと「ない」になる。
- **キャッシュを合わせる** = 音声をしまうときの「名前のつけ方」と、探すときの「名前の言い方」を **同じルール** にすること。

---

## ② 「名前」は何で決まる？（キーの中身）

1つの音声の「名前」は、次の **5つ** を並べたものだよ。

| 番号 | なに | いまのルール（例） |
|------|------|---------------------|
| 1 | **テキスト** | 英語の文（前後の空白とり、小文字、改行は \n にそろえる） |
| 2 | **言語** | 英語なら `en-US`、日本語なら `ja-JP` |
| 3 | **声** | 話者Aなら `alloy`、話者Bなら `nova` |
| 4 | **速さ** | ふつう `1`、ゆっくり `0.6` など |
| 5 | **モデル** | いまは `tts-1`（変えると名前がぜんぶ変わる） |

この5つが **ぜんぶ同じ** なら「同じ名前」→ キャッシュが **合っている**。  
1つでも違うと「別の名前」→ **合っていない**（再生されない）。

---

## ③ どこを「同じ」にすればいい？

次の **3つの場所** で、上のルールが **同じ** になっていないといけない。

| 場所 | なにをしているか | どこを確認する |
|------|------------------|----------------|
| **アプリ** | 再生するときに「この名前で探す」 | `lib/services/tts_cache_key_util.dart`（モデル名）<br>`lib/config/tts_voice_config.dart`（A=alloy, B=nova） |
| **事前準備（prefill）** | 音声をたくさん作って箱に入れるときの「名前」 | `scripts/prefill_tts_assets.dart`（モデル・声・テキストの正規化） |
| **サーバー（Edge）** | 音声を作って箱に入れるときの「名前」 | `supabase/functions/tts_synthesize/index.ts`（MODEL・正規化・声） |

**あわせるコツ**:  
「**モデル名**」（`tts-1` か `tts-1-hd` か）を変えると、名前のルールがぜんぶ変わる。  
モデルを変えたら、**3か所とも同じモデル** にして、**prefill をやり直す**。

---

## ④ キャッシュを合わせる手順（やることリスト）

### ステップ1: モデル名をそろえる

- いま使うモデルを **1つ** 決める（例: `tts-1`）。
- 次の3か所が **同じ文字** になっているか確認する:
  1. **アプリ**: `lib/services/tts_cache_key_util.dart` の `kTtsModel = 'tts-1'`
  2. **prefill**: `scripts/prefill_tts_assets.dart` のコメントや、Edge に送る body（モデルは Edge 側で使うので、prefill は同じ Edge を使えばOK）
  3. **Edge**: `supabase/functions/tts_synthesize/index.ts` の `const MODEL = "tts-1"`
- 違っていたら、**3か所とも同じ** に直す。

### ステップ2: 声（voice）をそろえる

- 話者 **A** → `alloy`、話者 **B** → `nova` で、アプリ・prefill・Edge で **同じ** にする。
- アプリ: `lib/config/tts_voice_config.dart`（`voiceRoleA` / `voiceRoleB`）
- prefill: `scripts/prefill_tts_assets.dart` の `_voiceForRole`（A=alloy, B=nova）
- Edge は、prefill やアプリから「voice を指定」して呼ぶので、**呼ぶ側が同じ** なら合う。

### ステップ3: テキストの正規化をそろえる

- 「名前」に入れるテキストは、**空白をとる・小文字にする・改行を \n にそろえる** というルールが、アプリ・prefill・Edge で **同じ** になっていること。
- いまのコードは同じルールになっているので、**変えなければ** そのままで合っている。

### ステップ4: 変更したら prefill をやり直す

- **モデル** や **声のルール** を変えたら、名前の形が変わるので、**昔入れた音声は「別の名前」のまま** 残る。
- だから **prefill をもう一度実行** して、**新しい名前** で音声を入れ直す。
  ```
  dart run scripts/prefill_tts_assets.dart
  ```
- 終わったら、確認用の一覧を出して、再生できるか見る（`docs/TTS_音声DB確認手順_やさしい版.md` の手順）。

---

## ⑤ まとめ（こども向け一言）

- **キャッシュを合わせる** = 音声の「名前のつけ方」を、**アプリ・prefill・サーバーでぜんぶ同じ** にすること。
- **名前** = テキスト・言語・声・速さ・モデルの5つをそろえたもの。1つでも違うと「別の名前」になる。
- **やること**: モデル名と声（A=alloy, B=nova）を **3か所とも同じ** にして、変えたら **prefill をやり直す**。
- そうすれば「名前が合う」ので、DB に保存した音声がアプリで再生できるようになる。
