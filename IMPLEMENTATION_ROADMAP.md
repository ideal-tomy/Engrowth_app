## 実装ガイドライン

### ブランチ命名
```
feature/issue-{番号}-{機能名}
例: feature/issue-1-enhance-search
```

### コミットメッセージ
```
[種類] 簡潔な説明

詳細な説明

Closes #{Issue番号}
```

### コードレビュー
- 各PRは最低1人のレビューが必要
- デザイン変更はスクリーンショット必須
- パフォーマンス影響がある場合は計測結果を添付

## 進捗管理

各Phase完了時に以下を実施:
1. 動作確認
2. パフォーマンステスト
3. ユーザーテスト（可能な範囲で）
4. ドキュメント更新
