# 初期Issueリスト

GitHubリポジトリにpush後、以下のIssueを作成することを推奨します。

## リファクタリング系

### 1. 不要ファイルの削除
- **タイトル**: `[Refactor] 不要なセットアップガイドと一時ファイルの削除`
- **内容**: 
  - セットアップガイド（`*_SETUP.md`, `*_GUIDE.md`など）の削除
  - 一時的なSQLファイルの削除
  - CSVデータファイルの削除（機密情報を含む可能性）
- **ラベル**: `refactoring`, `cleanup`

### 2. READMEの改善
- **タイトル**: `[Refactor] READMEの改善とドキュメント整理`
- **内容**: 
  - READMEの内容を最新の状態に更新
  - セットアップ手順の簡素化
  - プロジェクト構造の説明を追加
- **ラベル**: `documentation`, `refactoring`

## フロントエンド系

### 3. UI/UXの改善
- **タイトル**: `[Feature] ホーム画面のUI改善`
- **内容**: 
  - カードデザインの改善
  - アニメーションの追加
  - レスポンシブデザインの対応
- **ラベル**: `frontend`, `ui/ux`, `enhancement`

### 4. 検索機能の強化
- **タイトル**: `[Feature] 単語検索機能の強化`
- **内容**: 
  - リアルタイム検索
  - 検索履歴の保存
  - 高度なフィルタリング
- **ラベル**: `frontend`, `enhancement`

### 5. ダークモード対応
- **タイトル**: `[Feature] ダークモードの実装`
- **内容**: 
  - テーマ切り替え機能
  - システム設定との連携
- **ラベル**: `frontend`, `ui/ux`, `enhancement`

## バックエンド系

### 6. 認証機能の実装
- **タイトル**: `[Feature] ユーザー認証機能の実装`
- **内容**: 
  - Supabase Authの統合
  - ログイン/ログアウト機能
  - ユーザープロフィール
- **ラベル**: `backend`, `enhancement`

### 7. 進捗管理の強化
- **タイトル**: `[Feature] 進捗管理機能の強化`
- **内容**: 
  - 学習統計の表示
  - 達成度の可視化
  - 学習履歴の保存
- **ラベル**: `backend`, `enhancement`

### 8. 画像アップロード機能
- **タイトル**: `[Feature] 画像アップロード機能の実装`
- **内容**: 
  - Supabase Storageの統合
  - 画像のアップロード/削除
  - 画像プレビュー
- **ラベル**: `backend`, `enhancement`

## データベース系

### 9. データベース最適化
- **タイトル**: `[Refactor] データベースクエリの最適化`
- **内容**: 
  - インデックスの追加
  - クエリパフォーマンスの改善
  - キャッシュの実装
- **ラベル**: `database`, `refactoring`

### 10. データバックアップ機能
- **タイトル**: `[Feature] データバックアップ機能の実装`
- **内容**: 
  - 定期バックアップ
  - データエクスポート
  - データインポート
- **ラベル**: `database`, `enhancement`

## テスト系

### 11. ユニットテストの追加
- **タイトル**: `[Feature] ユニットテストの追加`
- **内容**: 
  - モデルのテスト
  - サービスのテスト
  - プロバイダーのテスト
- **ラベル**: `test`, `enhancement`

### 12. 統合テストの追加
- **タイトル**: `[Feature] 統合テストの追加`
- **内容**: 
  - E2Eテスト
  - 画面遷移のテスト
  - API連携のテスト
- **ラベル**: `test`, `enhancement`

## 使用方法

1. GitHubリポジトリの「Issues」タブを開く
2. 「New issue」をクリック
3. 上記のIssueを1つずつ作成
4. 適切なラベルを付ける
5. 優先度を設定（必要に応じて）

## 優先順位の提案

### 高優先度
- Issue #1: 不要ファイルの削除
- Issue #2: READMEの改善

### 中優先度
- Issue #3: UI/UXの改善
- Issue #6: 認証機能の実装

### 低優先度
- Issue #5: ダークモード対応
- Issue #11: ユニットテストの追加
